## Zielbild (bindend)

Die neue Design-Sprache (aus Welle 1) wird auf ALLE übrigen Admin-Seiten
ausgerollt, damit die gesamte Oberfläche einheitlich wirkt. Jede Seite wird
in `<PageShell>` mit korrekter Zone gesetzt und nutzt die vorhandenen
Block-Komponenten.

## Regel für JEDE Seite in JEDEM Batch (ohne Ausnahme)

1. Seite in `<PageShell zone="…" zoneLabel="…" …>` einwickeln, Breadcrumb
   und Titel setzen. Bestehendes `<h1>`/Ad-hoc-Kopf durch `PageShell`
   ersetzen.
2. GENAU EINE Primäraktion (`primaryAction`-Slot). Weitere Aktionen in
   `ActionMenu`. Konkurrierende gleichrangige bunte Buttons auflösen.
3. Tabs → `Segments`. Filter → `Facets` oder `Toolbar`. Sammlungen →
   `ListRows`/`CollectionGrid`. Reine Datentabellen → `DataTable`.
   Editor-Seiten → `MasterDetailLayout` + `FormSection`. Kennzahlen →
   `Metrics` (NUR bei echten Werten, NIEMALS Nullen-Leiste).
   Leerzustände → `EmptyState`. Sekundär-Uploads/-Aktionen → `Whisper`.
4. KEINE Änderung an State, Props, API-Calls, Event-Handlern.
5. Sichtbares Roh-HTML (`<p>…`, `&lt;…`) korrekt rendern bzw. auf Text
   strippen — nirgends sichtbare Tags.
6. Nach JEDEM Batch: `npm run lint` für die geänderten Dateien. Bei
   Fehlern: STOPP, beheben, bevor der nächste Batch beginnt.

## VALIDIERUNG (alle PASS erforderlich)

Führe JEDEN Check aus, PASS/FAIL einzeln. Bei FAIL: STOPP, beheben.

1. Jede Seite aus der Zuordnungstabelle nutzt `PageShell` mit der
   VORGEGEBENEN Zone. Kein eigenes `<h1>`/Ad-hoc-Kopf mehr. PASS/FAIL,
   Liste aller Seiten mit ihrer Zone.
2. Jede umgestellte Seite hat GENAU EINE visuell dominante Primäraktion;
   konkurrierende bunte Buttons sind aufgelöst. PASS/FAIL.
3. Zonenfarben korrekt: Auswertung/Assessment=Türkis, Ressourcen=Bronze,
   Verwaltung=Graphit. PASS/FAIL.
4. Keine `Metrics`-Leiste voller Nullen irgendwo. PASS/FAIL.
5. Kein sichtbares Roh-HTML irgendwo. PASS/FAIL.
6. KEINE funktionale Regression: je ein vollständiger Workflow-Durchlauf
   pro Batch (Batch C besonders gründlich: Assessment-Detail alle
   Sektionen durchklicken + je eine Aktion). PASS/FAIL pro Batch.
7. Kandidatenportal unverändert. PASS/FAIL.
8. `npm run lint` — null Fehler. PASS/FAIL.
9. `npm run build` — null Fehler. PASS/FAIL.

**COMMIT (nur wenn ALLE PASS):**
`Design-Sprache Breiten-Rollout abgeschlossen und validiert`
