## SCHRITT 1 — Toggle-State und Layout-Anpassung in ShellClient

1. In `components/shell/ShellClient.tsx`:
   - State ergänzen:
     `const [aiPanelOpen, setAiPanelOpen] = useState<boolean>(() => { try { return localStorage.getItem("eds_ai_panel_open") !== "false"; } catch { return true; } })`.
     Default IMMER `true` (Panel standardmäßig offen).
   - Bei State-Änderung: `localStorage.setItem("eds_ai_panel_open", String(aiPanelOpen))`.
   - `<AIPanel>` nur rendern wenn `aiPanelOpen === true`.
   - Das Vier-Zonen-Layout (`grid-template-columns` o.ä.) MUSS auf
     `aiPanelOpen` reagieren: bei `false` entfällt die
     `--eds-ai-panel-width`-Spalte, der Content-Bereich füllt den
     freiwerdenden Platz. Kein abruptes Springen — CSS-`transition`
     auf `grid-template-columns` oder `width` mit
     `var(--eds-transition-slow)`.
   - `setAiPanelOpen` via React-Context oder Prop an `TopBar` weitergeben
     (kein globaler State-Manager, kein Zustand-Drilling durch mehr als
     zwei Ebenen — direkte Prop an TopBar ist der korrekte Weg).
