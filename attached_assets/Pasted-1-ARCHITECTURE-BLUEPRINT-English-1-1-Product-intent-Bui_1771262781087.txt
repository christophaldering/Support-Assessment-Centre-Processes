1) ARCHITECTURE BLUEPRINT (English)
1.1 Product intent
Build a multi-tenant, enterprise-grade Executive Diagnostics Platform with:
Separate role-based portals (Admin / Moderator / Observer / Project Assistant / HR-Client / Candidate)
Deep AI layer (ChatGPT-style assistance) across the system
Offline-first Observer experience (tablet/iPad)
Consent management (per data type and per feature)
Full audit trails, versioning, “time travel”
Reporting engine (DOCX/PDF/PPT)
Analytics & benchmarking across heterogeneous models via score normalization
Workspace theming editor (full theme editor per workspace)
Admin access via neutral landing page, plus workspace admin passwords
EU data residency target (production); prototype may run elsewhere but must be architected for EU-only hosting and optional on-prem.
1.2 Tech stack decision
Use:
Next.js + TypeScript (App Router)
PostgreSQL (dev can run on hosted DB; prod EU)
Prisma ORM
Auth:
Candidates: individual accounts (email + initial password; forced password change at first login)
Admin portals: neutral landing page protected by “Master Admin” password + per-workspace admin password
Internal users (moderator/observer/assistant/hr-client): accounts with email+password; plus later SSO (Entra ID)
Offline-first Observer: PWA + local encrypted storage + sync engine

Analytics & benchmarking across heterogeneous scales/models
Key requirement: Different competency models + different scales must still be comparable.
Approach:
Always store raw scores (truth)
Additionally compute Normalized Score per rating and per consolidated node:
Default: map to 0–100 using linear transform based on scale definition
Optional: Z-score standardization within a defined benchmark population
Optional: percentile ranks per target role (requires sufficient data)
For cross-model comparison:
Provide mapping layer: “Competency Tags/Taxonomy” (optional V1 but recommended)
Without taxonomy, comparisons happen at abstracted dimensions (e.g., leadership domain) or “custom benchmark sets” defined by admin
