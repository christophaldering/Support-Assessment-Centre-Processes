SPRINT: D2-FLYOVER — ROBUSTE GUIDED TOUR (für alle Login-Wege)

ZIEL:
Implementiere einen Flyover/Guided Tour im ARAG BDP Tool, der unabhängig vom Login-Weg funktioniert:
- Login über /anmeldung (Plattform-Session) ODER /w/arag/login (Workspace-Login) ODER BDP-intern
- Tour richtet sich nach Rolle (Admin vs Beobachter vs Teilnehmer)
- In DEMO kann Tour automatisch starten (1x), in LIVE niemals automatisch.

KONTEXT / IST-STAND:
- Next.js 14 App Router
- app/arag-bdp/layout.tsx existiert (Bottom-Nav/Sidebar + Hamburger)
- app/arag-bdp/bdp-context.tsx liefert useBdp() mit { role, isAdmin, environment, userCode, workspaceSlug }
- data-testid Konvention vorhanden
- DEMO environment existiert und ist hart verdrahtet für Dummy-User (kein LIVE switch)

--------------------------------------------
1) TOUR ENGINE (EIGENE, LEICHTGEWICHTIGE IMPLEMENTIERUNG)
--------------------------------------------
MUSS:
1. Erstelle eine kleine Tour-Komponente OHNE externe Lib (stabil, kein Dependency-Risiko):
   - Datei: app/arag-bdp/components/TourOverlay.tsx (use client)
   - Features:
     - dim backdrop (overlay)
     - popover box (title, text, step counter)
     - “Weiter”, “Zurück”, “Beenden”
     - optional “Nicht mehr automatisch anzeigen” (nur DEMO)
     - highlight target element per data-testid (outline ring gold)
   - Wenn target nicht gefunden:
     - zeige Popover zentriert (fallbackMode)
     - und setze Hinweis klein: “Hinweis: Element aktuell nicht sichtbar – Schritt wird trotzdem erklärt.”
2. Tour state:
   - currentStepIndex, running, lastError
   - keyboard: ESC beendet Tour
3. Styling:
   - ARAG: Gold (#FFD700) als Akzent, Hintergrund #FFFBF0, Text schwarz
   - keine übertriebene Animation; nur fade 150ms

--------------------------------------------
2) STEP DEFINITIONS (JSON, rollenspezifisch)
--------------------------------------------
MUSS:
4. Erstelle Step-Definitionen als reine Daten:
   - Datei: lib/arag-bdp-tour.ts
   - Exporte:
     - getTourSteps({ role, isAdmin, environment }): Step[]
   - Step Schema:
     {
       id: string,
       title: string,
       body: string (DE),
       targetTestId?: string,   // optional
       route?: string,          // optional - wenn Step eine Route braucht
       placement?: "top"|"bottom"|"left"|"right"|"center"
     }
5. Rollen:
   - Admin Tour (10 Schritte)
   - Beobachter Tour (8 Schritte)
   - Teilnehmer Tour (6 Schritte)
6. Inhalte müssen exakt euer Tool erklären (Punkte-only):
   - Navigation (Home/Sessions/Bewertung/Auswertung)
   - DEMO Hinweis (nur in DEMO)
   - Bewertung: 100 Punkte je Kriterium, Summe==100, Sponsorflag nur Transparenz
   - Admin: Sessions/Teams/TN/Assignments/Kriterien, Release, Tie-break, Export
   - Auswertung gesperrt bis Release (LIVE), im DEMO ggf. sichtbar
   - Individual Notes sind Zusatz, beeinflussen nicht Siegerlogik
   Ton: professionell, leicht pfiffig, aber nicht albern.
