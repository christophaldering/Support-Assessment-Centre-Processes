REPLIT PROMPT — Competency Taxonomy + Semantic Tagging + Fit-Score Rubric (for cross-model matching)

Objective
Add a “Competency Taxonomy & Semantic Tagging” layer that enables robust matching and benchmarking across:
- different competency models (different structures, names)
- different rating scales
- different exercise designs
This module must improve the Requirements → Exercise Matching engine by providing:
1) A canonical taxonomy (multi-level) of executive-relevant competency concepts
2) A tagging system to map:
   - CompetencyModel nodes → taxonomy tags
   - ExerciseLibrary items/variants → taxonomy tags
   - RequirementsAnalysis extracted needs → taxonomy tags
3) A fit-score rubric combining:
   - hard filters (language, level, timebox)
   - taxonomy overlap and coverage
   - semantic similarity (embeddings when available)
   - AI rationale (explainable)
4) UI tooling to manage taxonomy and accelerate tagging (AI-assisted)
5) Governance & versioning (taxonomy evolves, mappings must be versioned)

Tech/Stack
- Next.js + TypeScript + Prisma + Postgres
- Reuse existing AI layer + audit/versioning + permissions/consent checks

===========================================================
E) FIT SCORE RUBRIC (Core Matching Logic)
===========================================================

Implement Fit Score as a blended model with transparent components.
Compute per ExerciseCandidate (library item + selected variant) a score 0–100.

1) Hard Filters (gating, can set to “exclude” or “penalize”)
- Language compatibility:
  - If no matching language variant exists: exclude OR heavy penalty (config)
- Seniority/level match:
  - if mismatch: penalty
- Timebox constraints:
  - if exercise too long: penalty or propose “adapt”
- Compliance constraints:
  - if exercise restricted/deprecated: exclude

2) Coverage Score (taxonomy overlap)
Define:
- R = set of requirement tags with weights r_i
- E = set of exercise tags with weights e_j
Compute weighted overlap:
- overlap = sum over tags t in intersection(R,E) of min(r_t, e_t)
- req_total = sum over r_t
CoverageScore = overlap / req_total  (0..1)

3) Breadth vs Depth
- Breadth: number of distinct Level-2 tags covered
- Depth: whether high-priority tags (weight > threshold) are covered
Compute:
- DepthScore = proportion of high-priority requirement tags covered

4) Semantic Similarity (optional)
If embeddings available:
- create short “exercise summary embedding” and “requirements summary embedding”
- SimilarityScore = cosine similarity (0..1)
If not available, fallback to AI ranking and set SimilarityScore = null.

5) Fit Score composition (configurable)
FitScoreDefinition.configJson example:
{
  "excludeIfNoLanguage": true,
  "weights": {
    "coverage": 0.45,
    "depth": 0.25,
    "semantic": 0.20,
    "constraints": 0.10
  },
  "thresholds": {
    "use_as_is": 0.80,
    "adapt": 0.55
  }
}

Compute:
- ConstraintsScore (0..1) from penalties (level/time/compliance)
- Fit = 100 * (wC*Coverage + wD*Depth + wS*Semantic + wK*Constraints)

6) Classification
- If Fit >= use_as_is threshold: “use_as_is”
- Else if Fit >= adapt threshold: “adapt”
- Else: “create_new”
Additionally:
- If hard constraints fail but concept match high: classify “adapt” with constraints explanation.

7) Explainability (required)
For each recommendation, store:
- top matching tags
- missing high-priority tags
- constraints penalties
- semantic similarity (if used)
- AI rationale text (short)
Expose this in the UI “Why this recommendation?”

G) NORMALIZATION CONNECTION (optional but recommended)
===========================================================

Add optional mapping to support benchmarking across models:
- Encourage tagging of competency nodes to canonical tags.
- When comparing across different competency models, allow aggregation by canonical tag.
- Store normalized scores per canonical tag via weighted mapping:
  - canonicalScore(tag) = sum(nodeNormalizedScore * nodeTagWeight) / sum(weights)

This enables cross-model benchmarks even with different structures.
